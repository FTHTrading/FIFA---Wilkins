Local Engines

On-Premise Rendering Stack

Three local engines handle the heavy lifting — AI image generation, 3D scene rendering, and video encoding — all running on-device with GPU acceleration.

ComfyUI

AI Image Generation

Template-driven workflows with queue-based execution. The orchestrator submits prompts to ComfyUI's API, polls queue and history for completion, and retrieves generated images.

InterfaceREST API — queue & history polling
WorkflowsTemplate-driven, JSON-defined
AccelerationCUDA GPU (RTX 5090)

Remote Provider

Cloudflare Workers AI

On-demand AI generation at the edge. Image generation via Cloudflare's inference API, integrated into the orchestrator as a remote provider alongside the local engine stack.

API Endpoints

GET
/v1/providers/cloudflare/status
Health check — returns provider readiness and model availability.
POST
/v1/providers/cloudflare/run
Execute generation — sends prompt to Cloudflare Workers AI, returns result.

Security Model

🔒
Credentials via env vars only

Cloudflare API tokens and account ID are loaded from environment variables — never hardcoded, never committed.

💾
Binary outputs saved to disk

Generated images are saved to output/providers/cloudflare/ on the local filesystem.

📄
JSON outputs returned directly

Text and structured responses are returned inline via the API response body.

Activation

Start the Media Studio

Two steps to bring the media pipeline online — configure the Cloudflare provider, then launch the orchestrator.

1
Configure Cloudflare Workers AI 
.\scripts\setup-cloudflare-workers-ai.ps1
Sets up API tokens and account credentials in .env
2
Launch the Orchestrator 
python -m uvicorn apps.orchestrator.main:app --port 8400 --reload
Starts the media orchestrator on port 8400
3
Verify provider status 
GET /v1/providers/cloudflare/status
Confirm Cloudflare provider is connected and ready
4
Run generation 
POST /v1/providers/cloudflare/run
Submit prompts and receive generated media
